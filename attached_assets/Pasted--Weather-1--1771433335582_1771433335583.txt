📄 ТЕХНИЧЕСКОЕ ЗАДАНИЕ
Проект: Weather+ (умный сайт погоды)
1. 🎯 Цель проекта

Создать веб-сайт с прогнозом погоды, который:

показывает стандартную погоду (как у всех)

даёт конкретные решения пользователю (что делать, что надеть, когда выйти)

2. 👤 Целевая аудитория

обычные пользователи (ежедневная проверка погоды)

люди, планирующие день (прогулка, спорт, поездки)

путешественники (сравнение городов)

🔍 Search input

4. ⚡ Уникальные мини-фичи (CORE VALUE)
4.1 🧥 “Что надеть”
Описание:

Алгоритм даёт рекомендацию по одежде

Вход:

температура

ощущается

ветер

осадки

Выход:

текст:

“Лёгкая куртка + кроссовки. Зонт не нужен”

4.2 🟢 “Лучшее время выйти”
Описание:

Показывает лучшие часы дня

Логика:

анализ 24 часов

фильтр по:

дождь

ветер

температура

Выход:

временные интервалы:

15:00–18:00 — идеально

18:00–20:00 — нормально

после 20:00 — плохо

4.3 📊 “Индекс комфорта”
Формула (пример):
comfort = tempScore + windScore + rainScore + humidityScore

Выход:

число (0–100)

статус:

80–100 → комфортно

50–80 → нормально

<50 → плохо

4.4 ⚔️ “Сравнение городов”
Функции:

выбрать 2 города

сравнить:

температура

ветер

осадки

комфорт

Выход:

“Алматы лучше для прогулки: теплее и меньше ветра”

4.5 🤖 Мини-помощник
Функции:

отвечает на вопросы:

“Когда лучше выйти?”

“Будет ли дождь вечером?”

“Где лучше погода?”

Логика:

не AI магия, а обработка данных + шаблоны

7. ⚙️ Бизнес-логика
7.1 Алгоритм “что надеть”

Пример:

if temp < 5 → зимняя куртка
if temp 5–15 → лёгкая куртка
if temp > 20 → футболка

if rain > 50% → добавить "зонт"
if wind > 8 → добавить "ветровка"

7.2 Алгоритм “лучшее время”

убрать часы с:

дождём > 40%

ветром > 8 м/с

выбрать лучшие интервалы

10. 🧠 Важная мысль

Если ты просто покажешь:

“15°C, ветер 5 м/с”

— это никому не нужно.

Но если ты скажешь:

“Идеально выйти в 17:00. Возьми лёгкую куртку”

— это уже продукт.
